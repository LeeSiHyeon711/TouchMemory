## 테스트 결과 (모두 통과)
1. `python3 -m venv .venv && pip install -r requirements.txt` → 설치 성공
2. `PYTHONPATH=src python3 -c "from touchmemory.db import init_db; init_db('data/touchmemory.db')"` → `data/touchmemory.db` 생성 확인 (init_db 함수 자체가 open→executescript→commit→close 처리)
3. `sqlite3 data/touchmemory.db "PRAGMA journal_mode;"` → `wal` 반환 확인
4. `sqlite3 data/touchmemory.db ".tables"` → `memory_items settings usage_events` 3테이블 확인
5. Repository 순환 테스트(스크래치패드 임시 스크립트로 실행, 저장소에는 없음):
   - `create_item` → id 반환 확인
   - `get_item` → content 일치 확인
   - `update_item(is_done=True)` → `is_done=True`이면서 `persistent_reminder=False` 그대로 유지(두 값 독립) 확인
   - `list_items(status="todo")` → 방금 완료 처리한 항목 제외 확인, `status="all"`은 포함 확인
   - `delete_item` → True 반환 + 이후 `get_item` None 확인, 존재하지 않는 id는 False 반환 확인
   - `add_event` → usage_events에 1행 적재(event_type/channel 값) 확인
   - `get_setting`/`set_setting` → 최초 None, upsert 후 값 갱신 확인
6. 테스트 후 `data/`, `.venv/`, `__pycache__/` 모두 삭제(커밋 대상 아님, `.gitignore`로도 제외됨) — 저장소는 소스 코드만 남은 상태
